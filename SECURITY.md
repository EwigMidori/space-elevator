# Security Policy

## Supported Versions

The project is currently in early public development.

Security fixes, when needed, are expected to land on the latest `main` branch first.

## Reporting A Vulnerability

Please do not open a public issue for a suspected security problem involving credentials, tokens, local automation escape paths, or unintended destructive repository behavior.

Instead, report it privately through GitHub security advisories or direct maintainer contact on the repository owner account.

When reporting, include:

- the affected version or commit
- a minimal reproduction
- expected impact
- any mitigation already identified
